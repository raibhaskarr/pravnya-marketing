# Pravnya.com Screenshot Capture Guide

Date: 2026-07-01
Owner: Product Design / Frontend
Status: Draft

## Purpose

This guide defines the production screenshots required for Pravnya.com. Do not fabricate app UI. Do not use random mockups. Every screenshot should come from the real Pravnya application, using approved demo data that contains no real child, parent, professional, school, medical, or therapy information.

## Capture Standards

- Use approved demo accounts only.
- Use stable demo data with realistic but non-identifying names.
- Capture in light mode unless a row below explicitly says otherwise.
- Remove browser chrome, debug overlays, test banners, build labels, and local environment indicators.
- Use the same device family across the set unless a specific page needs tablet context.
- Preserve enough margin so the website mockup frame can crop naturally.
- Do not annotate screenshots inside the image unless requested below. Website components provide external callouts.

## Review Checklist

- All screenshots use approved demo data.
- No real child or family data appears.
- No medical, diagnostic, or therapy claim appears unless legally approved.
- Text remains readable at website display size.
- Screenshots align with the Website Design & Content Handbook.
- Product, Legal, Security, and Marketing have approved the final set.
